# 转念 / Shift Mind

[中文](README.md) | [English](README.en.md)

> 每个念头都有一千种不同的样子。  
> `转念` 帮你看到每一种，找到你最喜欢的那个。

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)
[![Codex Skill](https://img.shields.io/badge/Codex-Skill-blueviolet)](https://openai.com)
[![Language: Chinese](https://img.shields.io/badge/Language-Chinese-red.svg)](#)

`转念 / Shift Mind` 是一个面向 Agent / Codex 的技能包。  
它不是“润色器”，不是“文风同义词生成器”，也不是 AI 套壳改写。

它做的事更直接：

1. 先看穿一句话真正想说什么
2. 再用不同滤镜的 `worldview + sentence logic + metaphor system` 重写
3. 输出能直接发出去的短终稿

当前首发 5 个滤镜：

- 王小波滤镜
- 张潇雨滤镜
- 鲁迅滤镜
- 李诞滤镜
- 特德姜滤镜

## 示例

原句：

`一个想法不一定对：“出片文化”助长了女装越做越拉跨的邪风。`

### 王小波

> 女装越做越难看，未必是设计师突然集体失明，不过是大家都忙着服务“出片”，懒得服务人了。

### 张潇雨

> “出片文化”真正改坏的，也许不只是衣服，而是大家看待身体和美的方式。衣服不再是拿来穿的，而是拿来给镜头交差的。

### 鲁迅

> 所谓“出片”，无非是叫人先忘了自己是个活人，再努力把自己摆成一件可供传看的东西。女装越做越不堪，也就不足为奇了。

### 李诞

> 现在很多女装也不是拿来穿的，主要是拿来拍的。认真想想，衣服都开始只服务照片了，那它对人差一点，好像也挺符合流程的。

### 特德姜

> 如果一套服装系统的优化目标，从“穿着体验”转向“镜头表现”，那么设计结果对现实中的身体越来越不友好，几乎是必然的。

## 安装

### 方式一：让 Agent / Codex / 龙虾帮你装

如果你在用支持本地文件操作的 Agent，比如 Codex、龙虾之类，最省事的方式就是直接对它说：

```text
请把 shift-mind 安装到全局 skills 目录。
```

或者更明确一点：

```text
请把当前目录里的 shift-mind 复制到全局 skills 目录，并保留 README、LICENSE 和 references。
```

如果你已经把它放在本地某个路径，比如：

```text
D:\5.github\xiaosheng-skills\shift-mind
```

也可以直接说：

```text
请把 D:\5.github\xiaosheng-skills\shift-mind 安装到我的全局 Codex skills 目录。
```

### 方式二：手动复制安装

把整个目录复制到：

```text
C:\Users\你的用户名\.codex\skills\shift-mind
```

本项目当前环境中的示例路径：

```text
C:\Users\allen\.codex\skills\shift-mind
```

复制完成后，重启 Codex。

### 方式三：CLI / 命令行安装

#### Windows PowerShell

```powershell
Copy-Item -LiteralPath 'D:\5.github\xiaosheng-skills\shift-mind' -Destination "$HOME\.codex\skills\shift-mind" -Recurse
```

#### macOS / Linux

```bash
cp -R ./shift-mind "$HOME/.codex/skills/shift-mind"
```

如果目标目录已存在，建议先确认里面是不是旧版本，再决定要不要覆盖。

## 支持的用法

这个 skill 支持接近 CLI 的短命令写法，尽量短，尽量直给。

### 1. 指定单个滤镜

```text
鲁迅滤镜：一个想法不一定对：“出片文化”助长了女装越做越拉跨的邪风。
张潇雨滤镜：AI 产品现在很热闹，但比赛可能还没开始。
李诞滤镜：一个想法不一定对：“出片文化”助长了女装越做越拉跨的邪风。
```

### 2. 指定多个滤镜

```text
鲁迅，张潇雨滤镜：进入高中生也足以做独立研究的时代，仍然让他们用所有时间做题。
王小波，李诞，特德姜滤镜：一个想法不一定对：“出片文化”助长了女装越做越拉跨的邪风。
```

### 3. 推荐滤镜

```text
推荐滤镜：AI 产品现在很热闹，但比赛可能还没开始。
```

### 4. 全部滤镜

```text
全部滤镜：一个想法不一定对：“出片文化”助长了女装越做越拉跨的邪风。
```

如果用户不用这种 mini-CLI 写法，而是自然语言表达，比如：

> “请用张潇雨的方式重写这句话”

也应该自动推断到对应模式。

## 这不是普通改写

大多数所谓“文风改写”干的事很虚：

- 换几个词
- 加一点情绪
- 模仿几句口头禅
- 最后把原句写得更长、更空、更像 AI

`转念` 不走这条路。

它的默认流程是：

1. 先提炼 `原句真意`
2. 判断这句话适合哪个滤镜
3. 用对应滤镜的认知模型升维
4. 输出 3 个短终稿
5. 推荐最强版本

硬规则只有一句：

> 如果删掉一半字以后更好，原版本就是废话。

---

## 输出长什么样

默认输出结构：

1. `原句真意`
2. `使用滤镜`
3. `版本1`
4. `版本2`
5. `版本3`
6. `推荐版本`

默认约束：

- 每版 1-3 句
- 尽量不超过 120 个汉字
- 能直接发，不是草稿
- 不写成长段散文
- 不搞模仿秀

如果原句太弱、太空、只有口号没有判断，就不强行重写，而是走滤镜自己的 `fallback_mode`。

```text
shift-mind/
├── SKILL.md
├── README.md
├── README.en.md
├── LICENSE
├── agents/
│   └── openai.yaml
└── references/

        ├── wang-xiaobo.md
        ├── zhang-xiaoyu.md
        ├── luxun.md
        ├── li-dan.md
        └── ted-chiang.md
```

各文件职责：

- `SKILL.md`：主入口，定义调用方式和默认行为
- `README.md`：中文说明
- `README.en.md`：英文说明
- `agents/openai.yaml`：UI 元数据
- `common-rules.md`：共享规则，防止所有滤镜写成一个味
- `filter-selection.md`：什么时候该用哪个滤镜
- `demos.md`：横向示例，方便校准
- `profiles/*.md`：每个滤镜自己的 worldview、pattern、禁区和 fallback

## 设计原则

这个项目有几个死规矩：

1. **先升维，再落字**
   先搞清楚真意是什么，再谈风格。

2. **学 worldview，不学口癖**
   真正的滤镜不是“像谁说话”，而是“像谁那样看问题”。

3. **短终稿优先**
   好句子不是写长写满，而是写透。

4. **不适配就拒绝**
   一个有品位的工具，知道什么时候不该硬写。

5. **不要变成 parody**
   滤镜应该让句子更准，不是更像 cosplay。

## 适合谁

这个 skill 最适合：

- 写社交媒体短帖的人
- 想把一句普通话写得更有辨识度的人
- 想观察同一句话在不同 worldview 下会如何变化的人
- 做内容产品、表达产品、AI 创作工具的人

不适合：

- 想要一键生成长文的人
- 只想改语法的人
- 只想“像某名人一样说话”却不在乎真意的人

## 写在最后

照片可以加滤镜，文字当然也可以。  
但转念真正值钱的，不是“变好看”，而是“变得更像它真正应该成为的那句话”。

有时候我们缺的不是表达能力。  
只是还没找到那个更适合看这句话的镜头。
